# 萤石云并发数限制（免费版）问题解决方案

## 🎯 问题根源

当在单个摄像头时仍然触发"并发数达到上限"错误，主要原因分析：

### 原因 1: 前端短时间内发送多个并发请求
- 用户创建摄像头 → 设备列表更新 → `VideoCenter` 立即为新设备加载预览流
- `loadPreviewStream` 会调用 `getVideoStreamUrl`，向后端请求流地址
- 若多个请求同时发送，会导致多个连接同时向萤石云建立，超过免费版并发限制（通常 1-2 个）

### 原因 2: 缺乏前端缓存机制
- 即使后端有缓存，前端在短时间内也可能发送多个 API 请求
- 特别是在"点击新建摄像头"等快速操作时，前端没有办法立即使用缓存

---

## ✅ 已实现的双层缓存解决方案

### 第 1 层：后端缓存（55 分钟）
**文件**: `backend/app/services/video_service.py` (第 100-102, 770-806 行)

- 流媒体地址缓存在服务器内存中
- 相同 `video_id` 的请求在缓存有效期内直接返回缓存数据，不向萤石云发送新请求
- 缓存有效期: 3300 秒（比萤石云 token 有效期短）
- 环境变量配置: `VIDEO_STREAM_CACHE_SECONDS`

```python
# 缓存配置
STREAM_URL_CACHE: Dict[int, Dict[str, Any]] = {}  # 缓存存储
STREAM_URL_CACHE_LOCK = threading.Lock()  # 并发保护
STREAM_CACHE_VALID_SECONDS = int(os.getenv("VIDEO_STREAM_CACHE_SECONDS", "3300"))
```

### 第 2 层：前端缓存（55 分钟）⭐ 新增
**文件**: `frontend/src/api/videoApi.ts` (新增前端缓存实现)

**关键特性**:
1. **本地缓存**: 浏览器内存中缓存 55 分钟，避免向后端发送重复请求
2. **请求去重**: 同一个 `video_id` 的并发请求共享一个 Promise，避免重复调用
3. **智能锁机制**: 如果请求正在进行中，后续请求等待同一个 Promise 完成

```javascript
// 前端缓存工作流
getVideoStreamUrl(1) // 发起请求
getVideoStreamUrl(1) // 等待上面的请求
getVideoStreamUrl(1) // 等待上面的请求
// 第一个请求完成 → 所有三个调用都返回缓存数据
```

## ⚠️ 注意事项

1. **缓存失效**: 
   - 设备流地址改变后，缓存需等待过期（55分钟）或手动清除
   - 如需立即生效，可重启后端来清除服务器缓存

2. **多标签页共享**:
   - 前端缓存仅在单个标签页内有效
   - 不同标签页会各自缓存，不会相互影响

3. **离线设备**:
   - 如果设备离线，缓存的流地址仍然有效
   - 设备重新在线时，下次缓存失效后会自动更新

## 🎓 原理说明

### 为什么会出现并发数限制？

萤石云免费版的并发连接限制通常为 1-2 个。当前端发送多个 `getVideoStreamUrl` 请求时：

1. 每个请求都会建立一个 HTTP 连接到后端
2. 后端的每个 `get_stream_info` 调用会建立一个连接到萤石云
3. 如果 5 个并发请求同时发送，就会有 5 个萤石云连接
4. 超过限制 → "并发数达到上限"错误

### 缓存如何解决？

```
┌─────────────────────────────────────────┐
│ 前端缓存                               │
│ 100ms 内的重复请求直接返回缓存结果      │
└────────────┬──────────────────────────┘
             │ 缓存未命中
             ↓
┌─────────────────────────────────────────┐
│ 后端缓存                               │
│ 3300 秒内的重复请求直接返回缓存结果     │
└────────────┬──────────────────────────┘
             │ 缓存未命中
             ↓
┌─────────────────────────────────────────┐
│ 萤石云 API                             │
│ 真实请求，建立连接，获取流地址          │
└─────────────────────────────────────────┘
```
